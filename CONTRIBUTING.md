# Contributing

Thanks for helping improve osu! Tournament Helper.

1. Open an issue before making behavior changes.
2. Keep command wording, permissions, tournament logic, database behavior, and
   schedules unchanged unless the change fixes a documented bug.
3. Put public deployment settings in `config.json` and secrets in `.env`.
4. Run `npm run smoke` and `npm run test:ci` before submitting a pull request.
5. Do not commit credentials, databases, logs, or tournament participant data.

Questions about project direction should be addressed to
myksmyks@KELTournaments.
