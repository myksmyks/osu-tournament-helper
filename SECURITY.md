# Security Policy

Do not report credentials or personal data in a public issue.

Send security reports privately to max@keltournaments.com with:

- A description of the issue and its impact
- Reproduction steps
- Affected files or versions
- Any suggested mitigation

Before publishing a fork, rotate credentials that may have been exposed and
remove secret-bearing files from the complete Git history. SQLite databases may
contain Discord IDs, osu! usernames, and tournament data and must not be shared.
